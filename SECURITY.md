# Security Policy

If you discover a security vulnerability within `react-theming-engine`, please let us know immediately!

---

## 🛡️ Reporting a Vulnerability

Please do **NOT** open a public issue on GitHub for security vulnerabilities. Instead, report them privately:

1.  **Contact**: Email Abilash or the repository maintainers.
2.  **Details**: Provide a detailed description of the vulnerability, including steps to reproduce it.
3.  **Timeline**: We will respond to your report as fast as possible and keep you updated on the fix.

### What is a Vulnerability?

A security vulnerability is a weakness in our code that could allow an attacker to:
-   **Gain unauthorized access** to data or functionality.
-   **Execute arbitrary code** in a user's browser.
-   **Disrupt service** or bypass security controls.

---

## ✅ Supported Versions

We currently only support the latest version of `react-theming-engine`. If you discover a vulnerability, please ensure you are using the newest release before reporting.

| Version | Supported |
| :--- | :--- |
| **0.1.x** | Yes |
| **< 0.1.0** | No |
